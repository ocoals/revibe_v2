## 2. 장애별 대응 절차

### 🧠 Claude Haiku API 장애 [P1]

**트리거:** recreation_failed 이벤트 급증 / API 타임아웃

1. **확인** (0~5분): status.anthropic.com 확인
2. **판단** (5~15분): 일시적 vs 장기 장애. Anthropic Status 업데이트 확인
3. **즉시 조치** (15~30분): 룩 재현 기능 비활성화 + 앱 내 메시지 표시
4. **Fallback** (30분+): Ximilar API 전환 (사전 구현 시). 미구현이면 비활성 유지
5. **복구** (정상화 후 10분): 기능 재활성화 → 테스트 1회 → 공지 해제

**예방:** 재시도 3회 (exponential backoff), 타임아웃 15초, Fallback API 사전 연동

### 🗄️ Supabase 장애 (DB/Auth) [P0]

**트리거:** 앱 전체 에러, 로그인 불가, API 전부 500

1. **확인** (0~5분): status.supabase.com + Dashboard
2. **범위 파악** (5~10분): DB만? Auth만? 전체? — 에러 로그 판별
3. **즉시 조치** (10분): 풀스크린 장애 안내 UI
4. **소통** (10~15분): Supabase Support 티켓 + Discord
5. **복구** (정상화 후 30분): 앱 재시작 → 데이터 무결성 확인

**예방:** Pro 플랜 (SLA 99.9%), 중요 쿼리 로컬 캐시, Health Check

### 🖼️ Cloudflare R2 장애 [P1]

**트리거:** 이미지 로딩 실패, 옷장 빈 화면

1. **확인** (0~5분): cloudflarestatus.com + R2 콘솔
2. **즉시 조치** (자동): 플레이스홀더 이미지 표시, 기능은 계속 이용 가능
3. **소통** (10분): Cloudflare Support 티켓
4. **복구** (정상화 후 15분): CDN 캐시 퍼지 → 이미지 확인

**예방:** 로컬 캐시 사용, 썸네일 SQLite 저장

### 🎨 remove.bg API 장애 [P2]

**자동 Fallback:** remove.bg 실패 → rembg (HuggingFace) 자동 전환 (이미 코드에 구현)

### 📱 앱 크래시 급증 [P0/P1]

**트리거:** Sentry 크래시율 >2%

1. **확인** (0~10분): Sentry 스택트레이스, 영향 OS/디바이스 파악
2. **범위 판단** (10분): 특정 디바이스? 특정 화면? 전체? → P0 vs P1
3. **긴급 수정** (30분~2시간): hotfix 브랜치 → 수정 → 테스트 → merge
4. **배포**: Edge Function 이슈 = 즉시. 앱 이슈 = Expedited Review
5. **확인** (배포 후 2시간): Sentry 크래시율 모니터링

### 💳 결제/구독 이슈 [P1]

**트리거:** 구독 결제 실패, 프리미엄 접근 불가

1. **확인** (0~10분): Apple/Google 콘솔, 웹훅 로그
2. **즉시 조치** (10분): 프리미엄 기능 임시 유지 (24시간 유예)
3. **대응** (30분): 웹훅 재처리, 구독 상태 수동 동기화
4. **소통** (1시간): 영향 사용자 개별 사과 + 보상 (1주 무료)

## 3. 사용자 공지 템플릿

### 인앱 배너 (P1)

> 현재 [기능명] 이용이 원활하지 않습니다. 빠르게 복구 중이며, 다른 기능은 정상 이용 가능합니다. 불편을 드려 죄송합니다.

### 인앱 풀스크린 (P0)

> 일시적인 서버 점검 중입니다 🔧
> 현재 서비스 이용이 어렵습니다. 최대한 빠르게 정상화하겠습니다.
> 잠시 후 다시 시도해 주세요.
> [재시도]

### 인스타 스토리 (P0 장기화)

> ⚠️ ClosetIQ 서비스 안내
> 현재 서버 이슈로 이용이 어렵습니다. [예상 시간]까지 정상화 예정입니다.
> 복구되면 바로 알려드릴게요! 🙏

### 복구 완료

> ✅ 정상화 완료. [시간] 발생한 이슈가 정상화되었습니다. 이용에 불편을 드려 죄송합니다.

### 보상 안내 (P0 1시간+)

> 불편을 겪으신 분께 무료 룩 재현 3회를 추가 제공해 드립니다. 이미 계정에 반영되었습니다.

**공지 원칙:** 솔직하게, 구체적으로, 시간 제시, 진심으로, 보상 명시

## 4. 포스트모템 템플릿

P0/P1 장애 발생 시 복구 후 24시간 이내 작성.

### 1. 요약
- 장애 제목 / 심각도 / 발생·감지·복구 시각 / 총 다운타임 / 영향 범위

### 2. 타임라인
- 분 단위로 이상 신호 → 감지 → 1차 조치 → 원인 파악 → 수정 → 정상화 → 공지 기록

### 3. 근본 원인 (Root Cause)
- 5 Whys 분석

### 4. 영향 분석
- 영향 사용자 수 / 기능 / 데이터 손실 여부 / 수익 영향

### 5. 조치 항목
- [ ] 즉시: 재발 방지 긴급 조치
- [ ] 1주 내: 모니터링 강화
- [ ] 1개월 내: 근본적 아키텍처 개선

### 6. 교훈
- 잘한 점 / 개선할 점 / 행운이었던 점
