# AI 패션 옷장 관리 서비스 — 사업 기획서

> **"내 옷으로 인플루언서 룩을 재현하다"**
>
> 작성일: 2026년 2월 21일 · 버전: MVP 기획 (1인 개발자 관점)

---

## 1. Executive Summary

**핵심 문제:** 인플루언서 코디를 보고 옷을 계속 구매하게 되어 지출이 증가하지만, 정작 "옷은 많은데 입을 옷이 없다"는 만성적 불만이 반복된다. 영감을 얻는 행위 자체가 소비를 유발하는 구조에 갇혀 있다.

**솔루션:** 현재 보유한 옷을 촬영하면 AI가 코디 추천, 인플루언서 룩을 내 옷으로 재현하는 방법을 제시한다. 새 옷 구매 없이 스타일링 만족감을 제공한다.

**가치 제안:** "지출 절감"보다 "나만의 스타일 발견"으로 포지셔닝. 인플루언서 룩 재현 기능이 핵심 차별점.

**타겟:** 20대 초중반 여성 (성장 엔진) + 20대 후반~30대 초반 직장 여성 (매출 엔진)

**수익 모델:** 프리미엄 구독 (월 6,900원) + 제휴 커미션 + B2B 데이터 (장기)

**손익분기:** 사용자 5,000명 기준 구독자 67명 (전환율 1.3%) → 월 고정비 35만 원 커버

## 2. 경쟁사 분석

글로벌 옷장 관리 앱 시장에는 주요 3개 경쟁사가 존재한다.

| 경쟁사 | 국가/유저 | 강점 | 약점 |
|--------|----------|------|------|
| Whering | 영국, 900만+ 유저 | 소셜 기능 중심 | AI 추천 베타 수준, 한국어 미지원 |
| Cladwell | 미국, 100만+ 다운로드 | 캡슐 워드로브 철학 | AI 품질 낮음, 한국어 미지원 |
| Acloset | 한국, 400만 유저 | AI 패션 비서 | 무료 100개 아이템까지만, 스타일 추천 부족 |

### 2.1 경쟁사 공통 한계점 (우리의 기회)

- 옷 등록 허들이 너무 높음: 하나씩 사진 찍고 태그, 대부분 중간에 포기
- AI 코디 추천 품질이 낮음: 실제로 쓸 만한 수준이 아님
- "소비 줄이기" 표방하지만 결국 쇼핑 유도 구조
- 한국 시장 맞춤 서비스 부재: 무신사/에이블리/지그재그 연동 없음

## 3. 차별화 전략

### 3.1 차별점 1: "오늘 입은 옷 1벌부터 시작" 온보딩

기존 앱들은 "옷장을 먼저 채우세요"라고 요구하지만, 이것이 가장 큰 이탈 요인이다.

**수정된 온보딩 전략:**
1. 가입 직후 오늘 입은 옷 전신 사진 1장 촬영
2. 30초 만에 3개 아이템 자동 등록 (배경 제거 + 색상 추출 + 카테고리 탭 선택)
3. 즉시 핵심 경험 제공 (기본 코디 추천 또는 룩 재현)

**점진적 확장 방식:** 매일 코디 기록 (메인 확장 경로), 침대 위 멀티샷 (빠른 보충), 갤러리 선택 업로드 (옵션)

**핵심 철학:** "옷장 먼저 채우고 시작"이 아닌 "핵심 경험 먼저 주고 옷장은 쓰면서 채운다"

### 3.2 차별점 2: "내 옷으로 인플루언서 룩 재현"

이 기능이 경쟁사와의 가장 큰 차별점이자 서비스의 핵심 훅(Hook)이다.

**사용자 경험 흐름:**
1. **Step 1:** 레퍼런스 이미지 입력 (스크린샷/갤러리 선택 + 공유 버튼)
2. **Step 2:** AI 분석 — 아이템 감지 + 속성 추출 (카테고리, 색상, 스타일)
3. **Step 3:** 내 옷장 매칭 — 아이템별 유사도 매칭 (DB 쿼리 기반, AI 아님)
4. **Step 4:** 결과 제시 — 레퍼런스 | 내 옷 재현 나란히 비교
5. **Step 5:** 갭 분석 — 없는 아이템 표시 + 외부 검색 링크 제공

### 3.3 차별점 3: 한국 패션 생태계 최적화

무신사/에이블리/지그재그/29CM 연동, 퍼스널 컬러 시스템 반영, 한국 사용자 맞춤 UX/UI

### 3.4 차별점 4: 수익 모델 — "쇼핑"이 아닌 "스타일링 구독"

경쟁사들이 결국 쇼핑 유도로 수익을 내는 반면, 우리는 "꼭 필요한 1벌만 추천" 방식으로 신뢰를 쌓는다.

## 4. 타겟 고객 세분화

| 구분 | 세그먼트 A: 성장 엔진 | 세그먼트 B: 매출 엔진 |
|------|---------------------|---------------------|
| 페르소나 | 인스타 탐색탭 중독자 | 출근 코디 고민러 |
| 연령 | 20대 초중반 여성 | 20대 후반~30대 초반 직장 여성 |
| 페인 | 인플 코디 따라 사면 안 어울림 | 아침마다 옷 앞에서 15분 낭비 |
| 반응 포인트 | 룩 재현 기능에 강하게 반응 | 매일 코디 추천으로 시간 절약 |
| 역할 | 사용자 확보 + 바이럴 | 매출 + 리텐션 |
| 리스크 | 구독 결제 민감 | 지불 의향 가장 높음 |

**실행 전략:** 마케팅은 A에 집중 (틱톡/릴스 바이럴), 프로덕트는 B도 만족 (매일 코디 추천).

## 5. MVP 기능 우선순위

### Tier 1: "이것 없으면 앱이 아니다" (0~3개월)

| 기능 | 설명 | 기술 |
|------|------|------|
| 온보딩 | 전신 사진 촬영 → 배경 제거 + 색상 추출 + 카테고리 탭 | rembg + OpenCV |
| 룩 재현 | 레퍼런스 → 아이템 감지 → 속성 추출 → 옷장 매칭 | Claude Haiku API |
| 옷장 관리 | 목록 보기, 수동 추가, 삭제/수정 | 기본 CRUD |
| 갭 분석 | 없는 아이템 표시 + 외부 검색 링크 | 딜링크 방식 |

### Tier 2 (출시 후 1~3개월): 데일리 기록, 코디 추천, 프리미엄 구독

### Tier 3 (3~6개월): 코디 다양화, CPA 제휴, 쇼핑몰 OCR, 멀티샷

### Tier 4 (6개월+): 퍼스널 컬러, 시즌 리포트, B2B, 커뮤니티

## 6. 기술 스택

| 구성 요소 | 선택 | 비용 |
|----------|------|------|
| 모바일 앱 | **Flutter (Dart)** | 0원 (오픈소스) |
| API 서버 | Supabase Edge Functions (Deno/TS) | 무료 ~ 월 3만 원 |
| 데이터베이스 | Supabase PostgreSQL (Free → Pro) | 무료 ~ 월 3만 원 |
| 이미지 저장소 | Cloudflare R2 | 무료 10GB |
| AI 추론 | Claude Haiku API (종량제) | 건당 ~$0.001 |
| 배경 제거 | remove.bg API + rembg fallback | 월 50회 무료 |
| 색상 추출 | K-Means 클러스터링 (코드 로직) | 0원 |
| 인증 | Supabase Auth (카카오/Apple OAuth) | 무료 |
| 자체 GPU 서버 | 없음 (API로 대체) | 0원 |

> **Flutter 선택 근거:** Share Extension(인스타 공유 수신), 카카오 공식 SDK, 이미지 처리 성능이 결정적. 상세는 [PRD Section 5.6](PRD.md) 참조.
