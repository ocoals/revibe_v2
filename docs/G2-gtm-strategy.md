# G2. Go-to-Market Strategy

> ClosetIQ · v1.0 · 2026.02.22

---

## 1. 전략 총괄

### 핵심 질문: "첫 1,000명을 어떻게 데려올 것인가?"

> 92%의 스타트업이 실패하는 이유는 제품이 아니라 시장 진입(GTM)이다.

### 타겟 & 메시지

- **Primary 페르소나:** 20~30대 여성, 인스타/틱톡에서 패션 콘텐츠 소비, "인플루언서 옷 따라입기" 경험 있음
- **Secondary:** 미니멀리스트, 옷장 정리에 관심 있는 2030
- **Hook:** "인플루언서 코디, 내 옷장으로 따라입기"
- **Sub:** "AI가 내 옷장에서 찾아주는 코디 매칭"
- **CTA:** "지금 내 옷장으로 시작하기"

### GTM 3단계 전략

| Phase | 기간 | 목표 | 핵심 전술 |
|-------|------|------|----------|
| **Phase 0: 프리런치** | T-8주 ~ T-0 | 대기자 200명 + 가정 검증 | 랜딩페이지, 커뮤니티 시드, 클로즈드 베타 20명, 티저 숏폼 3개 |
| **Phase 1: 런치** | T-0 ~ T+4주 | 500 설치 / 30 DAU / 5 구독 | 앱스토어 소프트 런치, Product Hunt, 인플루언서 시딩 5명, 얼리버드 할인 |
| **Phase 2: 초기 성장** | T+4주 ~ T+12주 | 3,000 설치 / 150 DAU / 30 구독 | 틱톡/릴스 주 2회, 바이럴 루프 최적화, ASO, 커뮤니티 |

## 2. 프리런치 실행 계획

### 2.1 랜딩페이지 (T-8주)

**목표:** 출시 전 대기자 200명 확보 + 수요 검증

**페이지 구성:**
- Hero: "인플루언서 코디, 내 옷장으로 따라입기" + 앱 목업 이미지
- How it works: 3단계 (사진 찍기 → AI 분석 → 매칭 결과) 애니메이션
- Social proof: 베타 후기 (확보 후 추가)
- CTA: 이메일 입력 → "얼리버드 알림 받기"

**기술:** Next.js + Vercel (무료), Tally 이메일 수집, Vercel Analytics

### 2.2 시드 유저 확보 (T-8주 ~ T-4주)

| 채널 | 방법 | 목표 | 비용 |
|------|------|------|------|
| 인스타그램 DM | 패션 해시태그 활동 유저에게 DM | 50명 | 0원 |
| 패션 커뮤니티 | 에브리타임, 네이버 카페 설문 포스팅 | 30명 | 0원 |
| 지인 네트워크 | 카톡/인스타 스토리 공유 | 30명 | 0원 |
| 틱톡 티저 | '내 옷장 AI 분석 결과' 15초 영상 | 100명 | 0원 |
| 소규모 인플루언서 | 팔로워 1K~5K 계정 5명 베타 초대 DM | 시드 5명 | 0원 |

### 2.3 클로즈드 베타 (T-4주 ~ T-2주)

- **인원:** 20명 (타겟 페르소나 매칭)
- **배포:** TestFlight (iOS) / Firebase Distribution (Android)
- **피드백:** 카톡 오픈채팅 + 주 1회 설문
- **인센티브:** 정식 출시 시 프리미엄 3개월 무료
- **검증 목표:** H1 갤러리 사진 수, H2 만족도 3.5/5+, 크리티컬 버그 0, 온보딩 완료율 70%+

### 2.4 티저 콘텐츠 (T-2주)

| # | 제목 | 포맷 | Hook | 해시태그 |
|---|------|------|------|---------|
| 1 | 문제 공감 | 릴스 15초 | "인플 코디 따라입으려고 옷 5벌 샀는데 다 실패함" | #옷장정리 #코디추천 #OOTD |
| 2 | 기능 쇼케이스 | 릴스 30초 | "AI한테 내 옷장 맡겨봤더니..." | #AI패션 #클로젯아이큐 |
| 3 | Before/After | 릴스 15초 | "같은 옷장, 다른 코디" | #뭐입지 #패션AI |

## 3. 런치 실행 계획

### 3.1 주차별 런치 플레이북

**T-0 (런치 당일)**
- 06:00 앱스토어 공개 전환
- 07:00 랜딩페이지 CTA → 스토어 링크 변경
- 09:00 대기자 200명 이메일 발송
- 11:00 인스타 피드 + 스토리 + 릴스 동시 게시
- 12:00 카톡 오픈채팅 + 지인 공유 요청
- 종일: 실시간 모니터링 (크래시, 가입률)

**T+1일 ~ T+3일:** 크리티컬 버그 핫픽스, 초기 유저 DM 피드백 10명, 앱스토어 평점 요청 팝업, 두 번째 릴스

**T+1주:** Product Hunt 런치 (화요일 오전 9시 PST), 인디해커스/HN Show HN, 얼리버드 마감 카운트다운

**T+2주 ~ T+4주:** 인플루언서 시딩 5명, 틱톡/릴스 정규 시작 주 2회, 얼리버드 마감 → 정상가, 월간 성과 리포트 + 피벗 판단

### 3.2 Go/No-Go 체크리스트 (런치 전)

- ✅ 크래시율 < 1%, AI 성공률 > 95%, 앱스토어 심사 승인
- ✅ 개인정보처리방침/이용약관 앱 내 링크, Privacy Nutrition Label
- ✅ PostHog 이벤트 34개 검증, 실시간 대시보드 알림
- ✅ 랜딩페이지 → 스토어 링크, 런치 콘텐츠 3개, 이메일 템플릿
- ✅ 고객 문의 대응 채널 (이메일) 준비

### 4.2 틱톡/릴스 콘텐츠 전략 (P0)

**콘텐츠 카테고리:**
- 🎭 문제 공감 (30%): "인플 코디 따라입으려고 옷 5벌 샀는데 다 실패함"
- ✨ 기능 시연 (30%): "AI한테 내 옷장 보여줬더니 이걸 조합해줌"
- 👗 Before/After (25%): "같은 옷 다른 코디" 변신
- 💡 팁/교육 (15%): "옷 적게 사고 잘 입는 법 3가지"

**운영 규칙:** 주 2회, 오후 7~9시, 15~30초, Hook 3초 원칙, 한글 자막 필수

**KPI:** 조회수 X. 프로필 방문 → 링크 클릭 전환율 1%+에 집중.

## 5. 바이럴 루프 설계

### 5.1 자연스러운 공유 포인트

```
유저가 룩 재현 완료 → 결과: "87% 매칭! 🎉"
  ├→ 인스타 스토리 공유 (비교 이미지 + @closetiq 워터마크 + QR)
  │    → 스토리 본 친구 → 프로필 → 링크 → 앱스토어 → 설치
  └→ 카카오톡 공유 (미리보기 카드 + "나도 해보기" 딥링크)
       → 링크 탭 → 앱스토어 → 설치
```

### 5.2 K-Factor 목표

| 지표 | 초기 | 6개월 |
|------|------|-------|
| 공유율 (결과→공유) | 15% | 25% |
| 공유당 노출 | 50명 | 80명 |
| 노출→설치 | 2% | 3% |
| 설치→가입 | 60% | 70% |
| **K-Factor** | **0.09** | **0.42** |
