## 1. 설계 원칙

### 1.1 핵심 원칙 3가지

| 원칙 | 설명 | 적용 |
|------|------|------|
| **30초 원칙** | 가입부터 핵심 가치 경험까지 30초 이내 | 옷장을 채우는 것이 아닌, 경험을 먼저 준다 |
| **1-Tap 접근** | 모든 핵심 기능은 홈에서 1탭 이내 | 룩 재현, 옷장, 코디 기록 모두 바텀 탭으로 접근 |
| **비교 중심 UI** | 룩 재현의 핵심은 직관적 비교 | 모든 결과 화면은 "레퍼런스 vs 내 옷" 나란히 레이아웃 |

### 1.2 UX 톤 & 무드

- **긍정적 언어:** "없는 아이템" → "이 아이템이 있으면 완벽해요!"
- **가벼운 톤:** 패션은 즐거운 것. 무겁거나 사무적이지 않게.
- **성취감 피드백:** 매칭 성공 시 confetti 효과, 옷장 확장 시 축하 메시지
- **미니멀 UI:** 정보 과부하 없이, 한 화면에 하나의 핵심 액션

## 2. 정보 구조 (Information Architecture)

### 2.1 네비게이션: 바텀 탭 4개

```
┌─────────────────────────────────────────────┐
│                                             │
│              (현재 화면 콘텐츠)               │
│                                             │
├──────────┬──────────┬──────────┬────────────┤
│  🏠 홈   │  👗 옷장  │  ✨ 룩재현 │  👤 마이  │
└──────────┴──────────┴──────────┴────────────┘
```

**설계 근거:**
- 4탭 구성 — 5탭 이상은 터치 타겟이 좁아지고 인지 부하 증가
- "룩 재현"을 독립 탭으로 배치 — 핵심 차별점이므로 최대 노출
- FAB(Floating Action Button) 사용 안 함 — 탭 네비게이션이 더 직관적

### 2.2 화면 계층 구조

```
📱 앱 루트
├── 🔐 인증 플로우 (비로그인 시)
│   ├── S01. 스플래시
│   ├── S01. 웰컴 (가치 제안 슬라이드 3장)
│   └── S02. 소셜 로그인 (카카오/애플/이메일)
│
├── ⚡ 온보딩 플로우 (최초 1회)
│   ├── S03. 촬영 유도 ("오늘 입은 옷을 찍어보세요!")
│   ├── S04. 아이템 확인 (배경 제거 + 색상 + 카테고리)
│   └── 완료 → 첫 룩 재현 유도
│
├── 🏠 탭: 홈
│   ├── 오늘의 코디 추천 카드 (Tier 2)
│   ├── 빠른 액션 버튼 (촬영 / 룩 재현 / 기록)
│   ├── 내 옷장 미리보기 (가로 스크롤)
│   ├── 최근 룩 재현 히스토리
│   └── 프리미엄 업그레이드 배너 (무료 유저)
│
├── 👗 탭: 옷장
│   ├── 카테고리 필터 (가로 스크롤 칩)
│   ├── 아이템 그리드 (3열)
│   ├── → S07. 아이템 상세 (풀스크린)
│   ├── → S08. 아이템 추가 (카메라 → 등록)
│   └── 무료 한도 프로그레스 바 (30벌)
│
├── ✨ 탭: 룩 재현
│   ├── S09. 레퍼런스 입력 (갤러리/공유)
│   ├── S10. 분석 중 (로딩 + 진행 표시)
│   ├── S11. 결과 (나란히 비교)
│   ├── → S12. 갭 분석 (없는 아이템 + 외부 링크)
│   └── 히스토리 (과거 재현 목록)
│
└── 👤 탭: 마이
    ├── 프로필
    ├── S15. 코디 캘린더 (Tier 2)
    ├── S17. 구독 관리
    └── 설정 (알림, 계정, 개인정보)
```

## 3. 사용자 흐름도 (User Flows)

### 3.1 플로우 A: 온보딩 (가입 → 첫 경험)

**목표:** 30초 안에 핵심 가치 경험

```
앱 실행
  │
  ▼
S01. 스플래시 (1.5초 자동 전환)
  │
  ▼
S01. 웰컴 슬라이드 (3장, 스와이프)
  │  슬라이드 1: "내 옷으로 인플루언서 룩을" (가치)
  │  슬라이드 2: "30초면 옷장이 만들어져요" (편의)
  │  슬라이드 3: "AI가 코디를 완성해줘요" (기술)
  │
  ▼
S02. 소셜 로그인
  │  ├── 카카오 로그인 (원탭)
  │  ├── Apple 로그인 (원탭)
  │  └── 이메일 로그인
  │
  ▼ (최초 가입 시)
S03. 촬영 유도
  │  "오늘 입은 옷을 찍어보세요!"
  │  [지금 촬영하기] ← Primary CTA
  │  [갤러리에서 선택] ← Secondary
  │
  ▼
  기본 카메라 앱 실행 → 전신 촬영
  │
  ▼ (자동 처리 ~3초)
  배경 제거 (rembg) → 아이템 분리 → 색상 추출 (OpenCV)
  │
  ▼
S04. 아이템 확인
  │  "3개 아이템을 찾았어요! ✨"
  │  각 아이템: [배경제거 이미지] [카테고리 탭] [색상 자동표시]
  │  [옷장에 추가하기] ← Primary CTA
  │
  ▼
  ┌─────────────────┐
  │ 룩 재현 해볼까요?│ (선택)
  ├────────┬────────┤
  │  예    │ 나중에 │
  │  ▼     │  ▼     │
  │ S09    │ S05    │
  └────────┴────────┘
```

**핵심 UX 포인트:**
- 웰컴 슬라이드는 **스킵 가능** (우상단 "건너뛰기")
- 카카오 로그인 버튼을 **가장 위에** 배치 (한국 사용자 대부분)
- 촬영 유도 화면에 **사람 실루엣 가이드** 표시
- 아이템 확인에서 카테고리는 **탭 1번이면 선택** (드롭다운 아님)
- "옷장에 추가하기" 후 즉시 **룩 재현 유도** (첫 WOW 모먼트)

### 3.2 플로우 B: 인플루언서 룩 재현 (핵심 흐름)

**목표:** 레퍼런스 이미지 → 내 옷 재현 결과까지 10초 이내

```
S09. 레퍼런스 입력
  │  입력 경로 2가지:
  │  ├── A. 앱 내 갤러리 선택
  │  └── B. 외부 앱 공유 (Share Extension)
  │
  ▼
S10. 분석 중 (로딩)
  │  진행 표시:
  │  ✅ 아이템 감지 완료
  │  ✅ 색상/스타일 분석 완료
  │  ⏳ 내 옷장에서 매칭 중...
  │
  │  [Claude Haiku API 1회 호출 → JSON 응답]
  │  [매칭 엔진: DB 쿼리 + 점수 계산]
  │
  ▼ (~3-5초)
S11. 결과 비교
  │  ┌─────────────────────────────┐
  │  │ 레퍼런스     │  내 재현 ✨   │
  │  │ [이미지]     │  [이미지]     │
  │  ├─────────────────────────────┤
  │  │ 아이템별 매칭 상세           │
  │  │ ✅ 상의: 아이보리 니트 (92%) │
  │  │ ✅ 하의: 카키 슬랙스 (71%)  │
  │  │ ❌ 신발: 없는 아이템         │
  │  ├─────────────────────────────┤
  │  │ [이미지 저장]  [공유하기]    │
  │  └─────────────────────────────┘
  │
  ├── 매칭 성공 아이템 → 탭하면 S07 아이템 상세
  │
  └── 매칭 실패 아이템 → [찾기] 버튼
        │
        ▼
      S12. 갭 분석
        │  "이 아이템이 있으면 완벽해요!"
        │  [브라운 로퍼]
        │  ├── 🔍 무신사에서 찾기 (딥링크)
        │  ├── 🔍 에이블리에서 찾기 (딥링크)
        │  └── 🔍 지그재그에서 찾기 (딥링크)
        │
        └── "이 아이템 없이도 다른 조합이 가능해요"
            [대안 코디 보기] (Tier 3)
```

**핵심 UX 포인트:**
- 로딩 화면에서 **단계별 진행 표시** → 기대감 조성, 체감 대기시간 감소
- 결과는 반드시 **나란히 비교** (위아래 아닌 좌우)
- 갭 아이템은 부정적이지 않게 → **"있으면 완벽해요"** 긍정 표현
- 결과 이미지 **인스타 스토리 규격으로 저장** 가능 → 바이럴 루프
- Share Extension 지원 → 인스타에서 바로 공유 가능

### 3.3 플로우 C: 옷장 관리

```
S06. 옷장 그리드
  │  [카테고리 칩: 전체 | 상의 | 하의 | 아우터 | ...]
  │  [3열 그리드: 배경제거 아이템 이미지]
  │
  ├── 아이템 탭 → S07. 아이템 상세
  │     │  [카테고리] [색상] [스타일 태그]
  │     │  [착용 횟수] [마지막 착용일]
  │     │  [이 옷으로 코디 찾기] [수정] [삭제]
  │     │
  │     └── [이 옷으로 코디 찾기] → S09 (이 아이템 포함 조건)
  │
  ├── [+ 아이템 추가] → S08
  │     │  카메라 촬영 → 배경 제거 → 색상 추출
  │     │  카테고리 탭 선택 → 세부 카테고리
  │     │  스타일/핏/패턴 (선택)
  │     └── [옷장에 추가]
  │
  └── 아이템 롱프레스 → 다중 선택 모드
        └── [삭제] 또는 [일괄 태그 수정]
```
