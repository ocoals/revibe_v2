## 1. 핵심 검증 가설

> "가정(Assumption)을 사실(Fact)로 바꾸는 최소 비용의 실험"

### 1.1 가설-실험-판단 매트릭스

#### H1 [CRITICAL] — 갤러리 사진 수 (코딩 전 1주)

- **가설:** 타겟 사용자의 갤러리에 전신 코디 사진이 월 10장 이상 있다
- **왜 중요:** 전신 사진이 없으면 온보딩 자체가 불가. 서비스 존재 이유가 사라짐.
- **실험:** 타겟 페르소나 20명에게 갤러리 분석 요청 (대면 or 스크린샷 공유)
- **성공 기준:** 20명 중 15명(75%)이 월 10장+ → PASS
- **실패 시:** 침대 위 멀티샷(플랫레이) 모드를 MVP에 포함

#### H2 [CRITICAL] — 룩 재현 만족도 (베타 2주)

- **가설:** AI 룩 재현 결과가 만족도 3.5/5 이상
- **왜 중요:** 핵심 가치 자체. 결과가 쓸모없으면 재방문 0.
- **실험:** 베타 20명이 각 3회 이상 룩 재현 → 인앱 만족도 + 사후 인터뷰
- **성공 기준:** 평균 만족도 3.5/5+ AND '다시 쓰겠다' 70%+ → PASS
- **실패 시:** 프롬프트 튜닝 2주 + Ximilar API 병행 테스트

#### H3 [HIGH] — 실착률 (Phase 2)

- **가설:** 사용자가 추천 코디를 실제로 착용한다 (실착률 30%+)
- **실험:** Phase 2 코디 추천 출시 후 4주간 추적
- **실패 시:** 사용자 선호도 피드백 루프 추가

#### H4 [HIGH] — 한도 전환 (출시 후 4주)

- **가설:** 30벌 한도가 프리미엄 전환의 효과적인 트리거
- **실험:** 초기 100명의 옷장 크기 분포 + 한도 도달 후 행동 추적
- **성공 기준:** 한도 도달 사용자의 8%+ 결제 → PASS
- **실패 시:** 20벌/40벌 A/B 테스트

#### H5 [MEDIUM] — 공유율 (출시 후 2주)

- **가설:** 결과 공유 기능이 공유율 15%+
- **실험:** 결과 화면 공유 버튼 탭률 + 공유 완료율 추적
- **실패 시:** 공유 CTA 위치/문구 A/B 테스트

### 1.2 검증 타임라인

```
코딩 전 (1주)    베타 (2주)       출시 후 4주      Phase 2 (4주)
 H1 갤러리분석    H1 재확인        H4 한도 전환     H3 실착률
                 H2 만족도        H5 공유율
```

## 2. 유저 인터뷰 스크립트

### 인터뷰 원칙 (The Mom Test)

> "이 아이디어 좋아요?"라고 묻지 마세요. 대신 **과거의 구체적 행동**을 물어보세요. "마지막으로 인플 코디를 따라입으려 한 적이 언제예요?" — 이게 진짜 데이터입니다.

### 2.1 Pre-Build 인터뷰 (H1 검증, 5~10명)

**대상:** 20~30대 여성, 인스타그램 패션 콘텐츠 팔로우 중
**시간:** 20~30분 / 방법: 카페 대면 또는 카카오 음성통화
**인센티브:** 스타벅스 기프티콘 5,000원

**A. 아이스브레이커 (2분)**
- Q1. 요즘 패션이나 옷에 관심이 많으신 편인가요?
- Q2. 인스타그램에서 패션 관련 계정을 팔로우하시나요? 대략 몇 개 정도?

**B. 현재 행동 파악 (10분)**
- Q3. 마지막으로 '이 코디 따라입고 싶다'고 느낀 게 언제인가요? 그때 어떻게 하셨어요?
- Q4. 그 코디를 따라입으려고 실제로 뭔가 한 적이 있나요?
- Q5. 평소 아침에 옷을 고르는 데 얼마나 시간이 걸리나요?
- Q6. '뭐 입지?' 하고 고민한 마지막 경험을 말씀해 주세요.
- Q7. 옷장을 정리하거나 관리하는 나만의 방법이 있나요?

**C. 갤러리 사진 확인 — H1 직접 검증 (5분)**
- Q8. 혹시 갤러리에 코디 사진이나 전신 사진을 찍어두시는 편인가요?
- Q9. 대략 한 달에 몇 장 정도 찍으시나요?
- Q10. 갤러리에서 최근 한 달간의 전신/코디 사진을 같이 세어봐도 될까요? (동의 시)
- **[실제 장수 기록: ___ 장/월] ← H1 핵심 데이터**

**D. 문제의 심각성 (5분)**
- Q11. 인플루언서 코디를 따라입으려다 포기한 적이 있나요? 왜?
- Q12. 이 문제를 해결하려고 써 본 앱이나 서비스가 있나요?
- Q13. '내 옷장에서 비슷한 코디를 AI가 찾아준다'면 얼마나 유용할 것 같으세요? (1~5)
- Q14. 이런 서비스에 월 얼마까지 낼 수 있을 것 같으세요?

**E. 마무리 (3분)**
- Q15. 옷장/코디 관련 '이런 게 있으면 좋겠다' 하는 게 있으세요?
- Q16. 베타 테스트에 참여하실 의향이 있으신가요?
- Q17. 주변에 비슷한 고민을 하는 친구가 있을까요?

### 2.2 Post-Beta 인터뷰 (H2 검증, 5명)

**대상:** 베타 2주간 룩 재현 3회+ 사용한 활성 유저

- Q1. 룩 재현 기능을 처음 써봤을 때 어떤 느낌이었나요?
- Q2. 결과가 '쓸 만하다'고 느낀 적이 있나요? 어떤 점에서?
- Q3. 결과가 '별로다'라고 느낀 적이 있나요? 어떤 점에서?
- Q4. 가장 잘 매칭된 경험과 가장 못 매칭된 경험을 하나씩 말씀해 주세요.
- Q5. 실제로 매칭 결과대로 옷을 입어본 적이 있나요?
- Q6. 옷장에 등록하는 과정은 어떠셨나요? 귀찮은 점?
- Q7. 이 앱을 친구에게 추천하시겠어요? 0~10점 중? (NPS)
- Q8. 돈을 내고 쓸 만큼의 가치가 있나요? 얼마까지?
- Q9. 가장 개선되었으면 하는 점 1가지?

## 3. 사용성 테스트 계획

> 5명이면 80%: Jakob Nielsen 연구에 따르면 사용성 테스트 5명이면 문제의 80%를 발견합니다.

### 3.1 테스트 시나리오 (Task-based)

| ID | 태스크 | 시나리오 | 성공 기준 | 심각도 |
|----|--------|---------|----------|--------|
| T1 | 온보딩 완료하기 | "앱을 설치했습니다. 회원가입 후 아이템 3개 이상 등록하세요." | 3개 등록 완료 | CRITICAL |
| T2 | 룩 재현 시도 | "이 코디 사진으로 룩 재현을 해보세요." | 결과 화면 도달 | CRITICAL |
| T3 | 결과 공유 | "결과가 마음에 듭니다. 친구에게 공유해 보세요." | 공유 시트/이미지 저장 | HIGH |
| T4 | 아이템 찾기 | "등록한 검정 바지를 찾아보세요." | 해당 아이템 찾기 | MEDIUM |
| T5 | 갭 아이템 쇼핑 | "없는 아이템을 살 수 있는 곳을 찾아보세요." | 외부 링크 클릭 | MEDIUM |

### 3.2 SUS (System Usability Scale)

10개 질문, 각 1~5점. 목표 SUS 점수: **68+ (평균 이상)**, 80+ = 우수, 90+ = 최고.

## 4. 베타 테스트 운영 계획

| 항목 | 내용 |
|------|------|
| 인원 | 20명 (타겟 페르소나 매칭) |
| 기간 | 2주 |
| 플랫폼 | iOS 우선 (TestFlight) |
| 인센티브 | 프리미엄 3개월 무료 |

### 4.1 2주간 운영 일정

- **Day 0:** TestFlight 초대, 웰컴 메시지, 설치 가이드
- **Day 1~3:** 온보딩 독려, 첫인상 미니 설문, 크리티컬 버그 대응
- **Day 4~7:** 룩 재현 3회 독려, 중간 설문 (만족도+버그), 피드백 DM
- **Day 8~11:** 자유 사용, 데이터 분석, 비활성 유저 리마인드
- **Day 12~14:** 최종 설문 (SUS+NPS), 1:1 인터뷰 5명, 종료 안내

### 4.2 수집 데이터

**정량:** 온보딩 완료율 70%+, 인당 아이템 10개+, 인당 룩 재현 3회+, D7 리텐션 50%+, 크래시율 <1%, AI 성공률 95%+

**정성:** 만족도 3.5/5+, NPS 30+, SUS 68+, "다시 쓰겠다" 70%+, WTP 중앙값, Top 3 개선/좋은 점

## 5. Go / No-Go 판단 기준표

| 지표 | 🟢 GO | 🟡 PIVOT | 🔴 STOP |
|------|-------|---------|---------|
| H1: 갤러리 사진 수 | 75%+가 월 10장+ | 50~75% | 50% 미만 |
| H2: 룩 재현 만족도 | 3.5/5+ | 2.5~3.5 | 2.5 미만 |
| 온보딩 완료율 | 70%+ | 50~70% | 50% 미만 |
| D7 리텐션 (베타) | 50%+ | 30~50% | 30% 미만 |
| NPS | 30+ | 0~30 | 0 미만 |
| "다시 쓰겠다" | 70%+ | 50~70% | 50% 미만 |
| 크래시율 | < 1% | 1~3% | > 3% |
| 크리티컬 UX 이슈 | 0~1개 | 2~3개 | 4개+ |

### 판단 규칙

- **🟢 GO:** H1, H2 모두 GO + 나머지 6개 중 4개+ GO → 앱스토어 심사 제출
- **🟡 PIVOT:** H1/H2 중 하나가 PIVOT → 원인 분석 → 2주 개선 → 2차 베타 10명
- **🔴 STOP:** H1/H2가 STOP 또는 3개+ STOP → 유저 인터뷰 10명 추가 → 재설계/중단
